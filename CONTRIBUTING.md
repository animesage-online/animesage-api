# Contributing to Animesage API

Thank you for considering contributing to the Animesage API! This guide will help you get started. If you have any questions or suggestions, please open an issue or join our community discussions.

## Prerequisites

To contribute to the Animesage API, you should be familiar with the following technologies:

- Node.js
- JavaScript
- Axios
- Cheerio

## Requirements

- Node.js v22.11.0+
- npm v10.9.0+
- MariaDB Server v10.5.24+

## Cloning the Repository

1. **Fork the Repository:**
   - Click the "Fork" button on the top right of the repository page.

2. **Clone Your Fork:**
   - Clone your fork to your local machine (make sure to change `<your-username>` to your GitHub username):
     ```bash
     git clone https://github.com/<your-username>/animesage-api.git
     ```

3. **Create a New Branch:**
   - Create a new branch for your changes:
     ```bash
     git checkout -b <new-branch-name>
     ```

## Updating the Codebase

1. **Follow the Project Structure:**
   - Ensure that any new code or modifications adhere to the existing project structure. This helps maintain consistency and readability across the codebase.

2. **Update Documentation:**
   - Ensure all changes are reflected in the documentation.

3. **Commit Changes:**
   - Follow the commit message guidelines below.

## Commit Message Guidelines

When committing changes, use the following format:

- **Prefix Types:**
  - `feat`: A feature, possibly improving something already existing
  - `fix`: Fixing a bug or issue
  - `optimize`: Improving performance or efficiency
  - `refactor`: Refactoring a specific section of the codebase
  - `docs`: Documentation changes

- **Examples:**
  - `feat: New anime search feature`
  - `fix: Resolve issue with anime title search`
  - `optimize: Improve response time for trending anime`
  - `refactor: Refactor anime details retrieval`
  - `docs: Update contributing guide`

Thank you for contributing to the Animesage API!
